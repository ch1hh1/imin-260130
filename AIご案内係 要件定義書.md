# AIご案内係 要件定義書（たたき台）

> 総務・受付・社内ヘルプデスクの一次対応を、生成AIによって変革する——  
> 属人化を排し、応対品質を平準化する。それが本サービスの目的である。

---

## 1. サービス概要

### 1.1 サービス名
**AIご案内係**（仮称）

### 1.2 サービスコンセプト
- **対象**：B2B（企業向け）カスタマーサポート・社内問い合わせ対応
- **目的**：問い合わせ内容に対する回答の自動生成により、総務・受付・社内ヘルプデスク等の**一次対応を効率化**する
- **差別化**：顧客企業固有の**規程・マニュアル・FAQ・業務手順**等を参照し、回答精度を高める
- **成果**：**属人化の排除**と**応対品質の平準化**の実現

### 1.3 想定ユーザー
| 区分 | 説明 |
|------|------|
| 導入企業 | 総務・人事・情報システム部門等、社内問い合わせの窓口を抱える企業 |
| エンドユーザー | 社員（制度・手続き・システム等に関する問い合わせを行う者） |
| 管理者 | AIご案内係の設定・ナレッジ管理・運用監視を行う担当者 |

---

## 2. 機能要件

### 2.1 コア機能

#### 2.1.1 問い合わせ受付・回答生成
| 項目 | 内容 |
|------|------|
| 入力形態 | テキスト（チャット形式）、メール転送、Webフォーム等を想定 |
| 処理フロー | 問い合わせ → 意図・トピック分類 → ナレッジ検索 → 回答文生成 → 提示（＋必要に応じて人間へエスカレーション） |
| 回答の根拠 | 顧客企業が登録した規程・マニュアル・FAQ・業務手順等に基づく引用・要約 |
| 品質担保 | 参照元ドキュメントの明示、信頼度スコア、不明時は「回答不可」または人間案内 |

#### 2.1.2 ナレッジベース管理
| 項目 | 内容 |
|------|------|
| 登録対象 | 就業規則、各種規程、マニュアル、FAQ、業務手順書、Q&A集等 |
| 形式 | PDF、Word、Excel、テキスト、Markdown、社内Wiki等の取り込み |
| 更新 | バージョン管理、定期・随時更新、差分反映 |
| スコープ | 企業ごとにテナント分離（他社データ参照なし） |

#### 2.1.3 エスカレーション・人的対応との連携
| 項目 | 内容 |
|------|------|
| 条件 | 信頼度が閾値以下、機密・苦情・例外対応が必要な場合等 |
| 動作 | 担当者への転送、チケット起票、 SLA に基づく対応フローとの連携 |
| 履歴 | 問い合わせ内容・AI回答・エスカレーション理由・対応結果の記録 |

### 2.2 管理・運用機能

| 機能 | 概要 |
|------|------|
| ダッシュボード | 問い合わせ数、解決率、エスカレーション率、人気トピック等の可視化 |
| ナレッジ編集 | 参照ドキュメントの追加・削除・更新、FAQの手動登録・修正 |
| プロンプト・閾値設定 | 回答トーン、長さ、エスカレーション条件等のカスタマイズ |
| ログ・監査 | 問い合わせログ、回答ログ、参照元のトレーサビリティ、監査用エクスポート |
| 権限管理 | 管理者・編集者・閲覧者等のロールとアクセス制御 |

### 2.3 セキュリティ・コンプライアンス

| 項目 | 方針 |
|------|------|
| データ保存場所 | 国内リージョン優先、顧客指定のクラウド/オンプレ可否の検討 |
| 暗号化 | 保存時・通信時の暗号化（TLS、保存データの暗号化） |
| アクセス制御 | 認証（SSO/MFA対応）、IP制限、ロールベースアクセス |
| 個人情報・機密 | マスク処理、最小限の保持期間、削除要請への対応手順 |
| 監査 | 操作ログ、参照ログの保持期間と第三者監査対応 |

## 3. 非機能要件

### 3.1 性能
- **応答時間**：一次回答まで 5〜15 秒程度を目安（ナレッジ量・モデルに依存）
- **同時接続数**：契約プランに応じた同時利用ユーザー数の保証
- **可用性**：目標 99.5% 以上（SLA として明文化を推奨）

### 3.2 拡張性
- ナレッジドキュメント数・サイズの増加に対応できるアーキテクチャ
- 企業数（テナント数）の増加に対応するマルチテナント設計

### 3.3 運用
- 障害検知、アラート、復旧手順の整備
- ナレッジ更新・モデル更新に伴う再学習・再インデックス手順
- バックアップ・リストア手順

## 4. 参照情報の種類と優先度（例）

| 優先度 | 情報種別 | 用途例 |
|--------|----------|--------|
| 高 | 就業規則・人事規程 | 休暇・勤怠・制度の問い合わせ |
| 高 | 社内FAQ・Q&A集 | 頻出質問の標準回答 |
| 高 | 業務手順書・マニュアル | 申請フロー・システム操作 |
| 中 | 総務・経理マニュアル | 経費・備品・施設利用 |
| 中 | 情報システム利用ガイド | パスワード・アクセス・障害時 |
| 低 | 過去の問い合わせ事例（匿名化） | 回答品質の継続改善 |

## 5. 成果指標（KPI）案

| 指標 | 説明 | 目標例 |
|------|------|--------|
| 一次解決率 | AIのみで完結した問い合わせの割合 | 60〜80% |
| エスカレーション率 | 人間に回した割合 | 20〜40% |
| 平均応答時間 | 問い合わせから初回回答まで | 10 秒以内 |
| 顧客満足度 | エンドユーザー・担当者アンケート | 4.0/5.0 以上 |
| 問い合わせ削減率 | 導入前対比の窓口問い合わせ件数 | 20〜30% 削減 |

## 6. リスクと対策（たたき台）

| リスク | 対策例 |
|--------|--------|
| 誤回答・不適切回答 | 閾値設定、人間レビュー、フィードバックループでナレッジ・プロンプト改善 |
| 機密・個人情報の漏洩 | ナレッジの事前チェック、マスク化、利用規約・ガイドラインの明示 |
| ナレッジの陳腐化 | 定期更新フロー、変更検知による再取り込み |
| 過度な依存によるスキル低下 | 「AIは一次対応、最終判断は人」という運用ルールの徹底、トレーニング |
